# 🌿 Veridian Registry: Frontend Technical Specification This document outlines the architecture, file structure, and implementation details for the Veridian Blue Carbon Credit Registry frontend. ## 1. Project Overview A production-grade React application built with **Vite**, styled with **TailwindCSS**, and utilizing **Lucide React** for iconography. The system supports three distinct user roles with unique dashboards and permissions. ## 2. File Structure ``` veridian-registry/ ├── src/ │ ├── assets/ # Brand assets, logos, and ecosystem imagery │ ├── components/ │ │ ├── common/ # Shared components (Navbar, Sidebar, Buttons) │ │ ├── forms/ # Multi-step registration and submission forms │ │ ├── charts/ # Carbon analytics and ledger visualizations │ │ └── ui/ # Atom-level components (Badges, Modals, Inputs) │ ├── contexts/ # Role-based Auth and Web3 (MetaMask) State │ ├── hooks/ # Custom hooks for API and Blockchain interactions │ ├── pages/ │ │ ├── auth/ # Unified Login, NGO/Company Registration │ │ ├── ngo/ # Dashboard, Project Submission, Portfolio │ │ ├── admin/ # Dashboard, Approval Panels, Audit Logs │ │ └── company/ # Dashboard, Token Requests, Ledger │ ├── App.jsx # Routing and Layout Wrappers │ └── main.jsx ├── tailwind.config.js # Veridian Design System (Manrope, Teal/Ocean Palette) └── package.json ``` ## 3. Design System Tokens (Veridian) - **Typography:** Manrope (Headlines), Inter (Data/Body) - **Primary Palette:** `#006D77` (Deep Teal), `#83C5BE` (Mist), `#EDF6F9` (Clean Surface) - **Shape:** `Rounded-lg` (8px) for containers, `Rounded-full` for active states. - **Glassmorphism:** `backdrop-blur-xl bg-white/70` used for navigation components. ## 5. Development Instructions 1. **State Management:** Use `React Context` for user roles and `ethers.js` for MetaMask connection. 2. **Styling:** Strictly follow Tailwind classes as defined in the provided HTML screens to maintain the "Veridian" aesthetic. 3. **Icons:** Use `lucide-react` for all system icons.
